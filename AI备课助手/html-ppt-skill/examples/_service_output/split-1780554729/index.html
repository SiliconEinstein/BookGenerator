核心问题：为什么需要NLP？

  • 海量生物医学文献、病例、说明书处于非结构化状态
  • 人工阅读效率低，难以大规模检索与证据汇总
  • 需要将文本转化为可计算的结构化信息
  • 目标：实体识别 → 关系抽取 → 知识组织 → 智能决策

NLP基础任务：命名实体识别（NER）

  • 目标：识别文本中的实体边界与类型
  • 生物医学实体：药物、疾病、基因/蛋白、不良反应等
  • 输出：实体列表（文本、起止位置、类型、置信度）
  • 课堂工具：spaCy（领域模型）或 BERT（预训练/微调）
  • 常见挑战：边界过长/过短、缩写、同义词

NLP基础任务：关系抽取（RE）

  • 目标：判断实体对之间是否存在特定关系
  • 输出：关系类型与方向（三元组：实体1—关系—实体2）
  • 实现路径：规则/模式（触发词、依存句法）或模型（Transformer）
  • 核心要点：共现 ≠ 因果，需要证据句支撑
  • 必须处理：否定句、对照句、假设句

评估度量：如何衡量抽取质量？

  • 为什么需要：衡量质量、比较方法、指导迭代
  • NER评估：边界是否正确、类型是否正确（Precision/Recall/F1）
  • RE评估：关系是否正确、方向是否正确、是否漏检
  • 严格匹配 vs 宽松匹配：边界完全一致 vs 部分重叠即可
  • 关键：可追溯证据句与来源

抽取误差来源（常见且可复现）

  • 实体边界：过长/过短（如「阿司匹林片」误识别为「阿司匹林片剂量」）
  • 实体类型：药物/代谢物/商品名混淆，疾病与症状混淆
  • 同义词/缩写：acetylsalicylic acid vs aspirin
  • 关系误判：共现不等于关系（同一句出现但无因果）
  • 否定/假设/对照句容易误判方向或存在性

课堂实操：完整流程（45分钟验收）

  • 步骤1：运行NER，抽取≥5个实体（含类型标注）
  • 步骤2：完成RE，抽取≥2条关系（含方向、证据句）
  • 步骤3：填写结构化表格（实体表 + 关系表）
  • 步骤4：快速人工复核，识别明显错误
  • 步骤5：给出1条可执行的改进建议
  • 交叉复核：组间互换结果，指出对方1处可能误判

质检与改进意识

  • 抽取结果是「候选证据」，不是「确凿事实」
  • 改进方向：调整实体类型集合、加入否定词过滤规则
  • 改进方向：提高阈值、替换模型、增加同义词归一后处理
  • 任务对齐：先定义（实体类型集合、关系集合、输出格式）再抽取
  • 强调可追溯：每条关系必须附证据句与来源文献ID
