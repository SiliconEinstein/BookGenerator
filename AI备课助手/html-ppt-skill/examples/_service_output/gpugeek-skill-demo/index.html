核心概念：从文本到结构化信息

  • 目标：把非结构化生物医学文本 → 可计算的结构化结果
  • NER：识别实体边界 + 实体类型（药物/疾病/基因/不良反应等）
  • RE：判断实体对是否有关联 + 关系类型 + 方向
  • 典型输出：实体表、关系表、三元组（E1-Rel-E2）
  • 关键原则：先定义实体/关系集合与用途，再开始抽取

NER：怎么做、输出是什么

  • 输入：句子/段落（文献摘要、说明书、药物警戒文本等）
  • 方法：spaCy 领域模型 或 BERT 预训练/微调模型
  • 输出字段：实体文本、起止位置、类型、置信度/分数
  • 课堂要求：至少覆盖两类实体，总数≥5
  • 常见坑：边界过长/过短、类型混淆、缩写被误当独立实体

RE：怎么做、如何形成关系/三元组

  • 前提：给定实体（来自 NER 或人工标注）
  • 规则入门：共现 + 触发词/模式（causes/induces/inhibits/associated with）
  • 方向判定：A inhibits B 等动词触发 + 句法/位置规则
  • 输出建议：实体1|类型1|关系|实体2|类型2|证据句
  • 质量底线：每条关系必须可追溯到证据句

医学NLP应用：关系抽取 vs 不良反应抽取

  • 关系抽取：药物—靶点/疾病/通路/不良反应 → 支撑证据链与知识图谱
  • 不良反应抽取：药物—不良反应事件 → 汇总安全性信号
  • 不良反应常扩展字段：严重程度、时间、剂量、人群
  • 业务价值：检索与重定位、候选线索筛选、说明书信息整理
  • 对比要点：RE更泛化；不良反应抽取更强调事件要素与语义约束

评估度量：评什么、为什么、用来改什么

  • NER评估：边界对不对、类型对不对 → Precision/Recall/F1
  • RE评估：关系对不对、方向对不对、是否漏检 → Precision/Recall/F1
  • 严格/宽松匹配：边界或事件要素是否必须完全一致
  • 为什么评估：比较方法、定位错误来源、指导迭代
  • 改进指向：降低误检（提精度）或减少漏检（提召回）

误差来源与质检清单（必须会找）

  • 边界错误：实体截断/粘连（过短或过长）
  • 类型错误：药物/代谢物/商品名混淆；疾病 vs 症状
  • 同义词/缩写：aspirin vs acetylsalicylic acid 等需归一
  • 关系误判：共现≠关系；跨句误连
  • 否定/假设/对照：not/no/without 或对照组导致方向/存在性误判

课堂实操流程：跑通→提交→改进（17分钟核心）
