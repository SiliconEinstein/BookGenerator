Python 异步编程入门

什么是异步编程

  • 传统同步：代码逐行执行，等待 I/O 时阻塞
  • 异步模式：I/O 等待期间可执行其他任务
  • 适用场景：网络请求、文件读写、数据库查询
  • 核心优势：高并发下提升资源利用率

协程基础：async/await

  • async def 定义协程函数
  • await 关键字挂起当前协程，等待异步操作完成
  • 协程不会自动运行，需要事件循环调度
  • 示例：async def fetch(): data = await get_data()

事件循环：asyncio.run()

  • 事件循环是异步程序的核心调度器
  • asyncio.run(main()) 启动循环并执行协程
  • 循环管理所有协程的挂起与恢复
  • Python 3.7+ 推荐使用 asyncio.run()
  • 自动处理循环的创建、运行和关闭

Python 异步编程入门

并发任务：gather 与 create_task

  • asyncio.gather(*tasks) 并发执行多个协程，等待全部完成
  • asyncio.create_task(coro) 将协程封装为 Task 立即调度
  • gather 返回结果列表，保持顺序
  • create_task 适合后台任务，不阻塞主流程
  • 示例：results = await asyncio.gather(fetch1(), fetch2())

异步 I/O：文件与网络

  • 标准库：asyncio.open() 异步读写文件
  • 网络请求：aiohttp 库实现异步 HTTP 客户端
  • 数据库：asyncpg、aiomysql 等异步驱动
  • 注意：requests 库是同步的，需替换为 aiohttp
  • 异步 I/O 能显著提升高并发场景性能

实践建议

  • 优先使用高层 API：asyncio.run()、gather、create_task
  • 避免在协程中调用阻塞函数（如 time.sleep，改用 asyncio.sleep）
  • 混合同步代码时使用 runinexecutor 包装
  • 调试工具：asyncio.run(debug=True) 开启详细日志
  • 小步迁移：从单个模块开始异步化，逐步扩展
