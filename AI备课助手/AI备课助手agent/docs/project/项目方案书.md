# AI备课助手项目方案书

## 1. 项目背景

AI备课助手旨在面向教师备课场景，提供从“课程材料解析”到“教学内容生成”再到“教学演示文稿产出”的一体化能力。当前阶段项目聚焦于一个可被后端系统调用的 Agent 服务，将课程大纲 PDF 解析为结构化课程目录，并进一步为课程章节生成章节大纲和 PPT 内容，最终通过 API 方式对外提供能力。

项目第一阶段的核心目标不是做一个泛化的“全能教学平台”，而是先构建一个稳定、可迭代、可评估的课程内容生成 Agent 服务底座，为后续扩展多学科、多模板、多教学风格和多模态素材接入提供基础。

---

## 2. 项目目标

### 2.1 总体目标

建设一个面向后端调用的 AI Agent 服务，完成以下三类任务：

1. **课程大纲解析**：输入课程大纲 PDF，输出结构化课程目录 JSON。
2. **章节大纲生成**：基于课程目录，为每个章节生成章节大纲，至少包含以下四类内容：
   - 教学目标
   - 核心知识点
   - 建议教学结构
   - 其他补充信息
3. **章节 PPT 生成**：基于章节大纲生成课程章节 PPT，供教师进一步编辑和使用。

### 2.2 阶段目标

第一阶段重点达成以下结果：

- 建立清晰的输入/输出协议与 API 规范。
- 形成可落地的 Agent 工作流和模块边界。
- 支持单个课程从 PDF 到 JSON、到章节大纲、到 PPT 的完整链路。
- 保证结果具备基础稳定性、结构一致性和可复用性。
- 为后端系统提供异步任务式调用接口。

### 2.3 非目标范围

当前阶段暂不重点处理以下内容：

- 教学视频、音频、动画等多模态课件自动生成。
- 面向学生端的互动问答或作业系统。
- 教务排课、权限管理、课程运营后台等完整业务系统能力。
- 高度复杂的 PPT 可视化设计引擎。
- 所有学科、所有格式课程材料的完全兼容。

---

## 3. 核心业务场景

### 场景一：课程大纲 PDF 结构化解析

用户或后端系统上传课程大纲 PDF，系统自动识别课程名称、章节层级、课时安排、章节标题、章节说明等信息，并输出统一格式的课程目录 JSON，供后续流程复用。

### 场景二：课程章节大纲自动生成

系统针对每个课程章节，结合章节标题、上下文、课程定位及教学逻辑，自动生成标准化章节大纲，帮助教师快速完成备课初稿。

### 场景三：课程章节 PPT 自动生成

系统基于章节大纲生成教学 PPT，输出可继续编辑的 PPT 文件，用于教师二次修改和课堂展示。

---

## 4. 需求拆解

## 4.1 功能需求

### 4.1.1 课程大纲解析模块

**输入：**
- 课程大纲 PDF 文件

**输出：**
- 结构化课程目录 JSON

**核心能力要求：**
- 读取并提取 PDF 文本内容。
- 识别课程基础信息，如课程名称、总学时、课程简介（如存在）。
- 识别章节层级结构。
- 识别每章课时、教学要求、章节说明等可用信息。
- 对解析结果进行结构化标准化输出。
- 当 PDF 结构不规整时，具备一定纠错和补全能力。

### 4.1.2 章节大纲生成模块

**输入：**
- 课程基础信息
- 课程目录 JSON
- 指定章节信息

**输出：**
- 章节大纲 JSON

**必含内容：**
- 教学目标
- 核心知识点
- 建议教学结构
- 其他补充信息

### 4.1.3 章节 PPT 生成模块

**输入：**
- 章节大纲 JSON
- 可选模板参数
- 可选风格参数

**输出：**
- PPT 文件
- 可选中间结构化页面描述 JSON

---

## 5. 总体方案设计

建议采用“**Agent 编排层 + 基础能力模块 + API 服务层**”的分层架构。

- **API 服务层**：向后端暴露统一接口，负责请求接收、参数校验、任务创建、状态查询和结果返回。
- **Agent 编排层**：负责将业务任务拆分为多个步骤，组织执行流程，控制模型调用、重试、校验和后处理。
- **基础能力模块**：包括 PDF 解析、结构化抽取、章节大纲生成、PPT 页面规划、PPT 文件生成、文件存储等模块。

---

## 6. Agent 工作流设计

### 6.1 任务一：课程目录生成工作流

1. 接收文件并完成基础校验。
2. 使用 PDF 解析模块提取文本、页码信息和可能的段落结构。
3. 对原始内容做清洗、切分与章节候选识别。
4. 使用 **LLM 为主** 的方式抽取课程基础信息与章节结构（提示词：`src/agent/prompts/outline_parse.md`）；解析失败时回退规则引擎。
5. 将抽取结果映射到统一 JSON Schema。
6. 执行结构校验与必要修复。
7. 输出课程目录 JSON。

### 6.2 任务二：章节大纲生成工作流

1. 读取目标章节及其上下文信息。
2. 加载 `src/agent/prompts/section_generate.md` 构造 prompt。
3. 调用 LLM 生成章节大纲（微调使用 `section_refine.md`）。
4. 将生成结果映射为标准 JSON。
5. 校验字段完整性与内容质量。
6. 如质量不足，则进行补全或再次生成。
7. 输出章节大纲 JSON。

### 6.3 任务三：章节 PPT 生成工作流

1. 加载 `src/agent/prompts/ppt_plan.md`，由 LLM 生成幻灯片规划 JSON（`*.slides.json`）。
2. 为每一页生成标题、正文要点。
3. 使用 `python-pptx` 将规划 JSON 渲染为 `.pptx`。
4. 返回文件地址、任务结果与中间结构 `slides.json`。

---

## 7. API 设计建议

建议采用异步任务制接口。

- `POST /api/v1/course-outline/parse`
- `POST /api/v1/chapter-outline/generate`
- `POST /api/v1/ppt/generate`
- `GET /api/v1/tasks/{task_id}`
- `GET /api/v1/tasks/{task_id}/result`

---

## 8. 下一步建议

1. **确定统一数据 Schema**：课程目录、章节大纲、PPT 页面描述三类 JSON 结构先定稿。
2. **确定 API 契约**：先把输入输出协议稳定下来，再进入编码。
3. **搭建最小可运行骨架**：API 服务、任务系统、文件上传、日志模块先搭好。
4. **先打通任务一和任务二**：即 PDF -> 课程目录 -> 章节大纲，PPT 能力可在此基础上接入。
5. **将现有 PPT 代码收敛为独立模块**：后续作为第三阶段接入，避免当前设计被历史实验代码牵制。
