### 已可确认
- 第一阶段主链路：`课程基础结构生成 -> 课程结构确认与调整 -> PPT内容生成 -> PPT基础编辑 -> 扩展资源挂载 -> 导出PPT`
- 目标用户：偏 `高校教师` 场景
- 优先学科特征：偏 `工程类、数据类、强实践性课程`
- 课程目录层级：至少为 `章-节` 两层
- 章节大纲已明确字段：`教学目标`、`核心知识点`、`建议教学结构`
- 输出语言：至少支持 `中文/英文`
- PPT风格：至少支持 `学术简约风`、`现代科技风`、`专业商务风`

## 1. 基础范围

### 1.1 第一阶段目标
- 项目第一阶段目标是什么？请用一句话描述。
  - 基于课程文档生成可编辑、可扩展、可导出的标准化课程包，覆盖课程结构、章节内容与PPT课件生产流程。
- 第一阶段是否只做 `课程大纲 PDF -> 课程目录 -> 章节大纲 -> PPT` 这条链路？
  - 是的。虽然中间还包括 `课程结构确认与调整`、`PPT基础编辑`、`扩展资源挂载`、`导出PPT` 等环节，但这些并不是我们的agent需要处理的任务，我们只负责内容生成。

### 1.2 业务边界
- 当前项目主要面向哪些用户：`高校教师`、`职业教育`、`K12`、`企业培训`，还是其他？
  - `高校教师`
- 当前是否有明确优先支持的课程类型或专业方向？
  - 偏 `工程类、数据类、强实践性课程`

## 2. 输入材料

### 2.1 输入格式
- 第一阶段支持哪些输入格式：`PDF`、`DOCX`、图片扫描件、Markdown？
  - PDF、txt两种类型
- 上传方式是 `直接上传文件` 还是 `传文件 URL`，还是两者都支持？
  - 两种都支持
- 是否需要支持一次上传多个文件？
  - 不需要

### 2.2 PDF 特征
- 你们的课程大纲文件大多数是 `文本型 PDF` 还是 `扫描型 PDF`？
  - 文本型
- 第一阶段是否必须支持 `OCR`？
  - 直接调用 PDF 解析接口
- 如果 PDF 格式很不规范，是否接受解析效果下降但流程不中断？
  - 接受
